Stocky - Database Schema

Supabase Postgres schema documentado a partir das migrations locais. Atualizado em 2026-06-17.

Resumo

Dominio principal

Produtos, lotes, movimentacoes de estoque e perfis de usuario.

produtos lotes movimentacoes perfis

Leituras operacionais

A view estoque_atual agrega entradas e saidas por produto para alimentar mobile, alertas e dashboard.

estoque_atual alertas

IA persistida

ai_logs guarda insights estruturados para UI mobile/web consumir sem esperar agentes longos.

ai_logs jsonb RLS

Auth e RBAC

Supabase Auth cria usuarios; perfis guarda nome e papel. O backend tambem valida RBAC antes de acessar dados via service role.

auth.users operador admin

Mapa de Entidades

produtos

Catalogo e parametros de estoque.

  • PK: id
  • Unique: sku
  • Atualiza atualizado_em via trigger

lotes

Lotes por produto, validade e fornecedor.

  • PK: id
  • FK: produto_id -> produtos.id

movimentacoes

Entradas, saidas e ajustes de estoque.

  • PK: id
  • FKs para produto, lote e usuario
  • Base da view estoque_atual

perfis

Extensao de usuario Supabase para RBAC.

  • PK/FK: id -> auth.users.id
  • Role: operador ou admin

ai_logs

Insights estruturados de IA para operadores e admins.

  • PK: id
  • JSONB: conteudo, fontes
  • Filtros: tipo, audiencia, severidade

View Operacional

estoque_atual calcula a posicao atual por produto. Entradas somam, saidas subtraem e ajustes atualmente entram como zero na view.
