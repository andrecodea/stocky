Stocky — API Reference

FastAPI · Supabase JWT Auth · RBAC (Operator / Admin) · SSE Streaming · v1.0

BASE URLhttps://api.stocky.app/v1     AUTH HEADERAuthorization: Bearer <supabase_jwt>
All endpoints validate Supabase JWT tokens and enforce RBAC before executing. Agent outputs are consumed via GET /ai/insights (poll) or Supabase Realtime subscriptions on ai_logs — no agent-specific endpoints exist. The POST /webhooks/supabase endpoint is called only by Supabase internal webhooks; it is not authenticated with a user JWT.
public No auth required
any Any authenticated user
operator Operator or Admin
admin Admin only
internal Supabase webhook secret

POST
/ai/chat
any
RAG Copilot chat. Streams SSE response via pgvector + LLM (OpenRouter).
Request Body
{
  "message": "Quais produtos estão em risco de ruptura essa semana?",
  "session_id": "uuid"    // optional, for context continuity
}
Response — Server-Sent Events (text/event-stream)
data: {"delta": "Com base nos dados atuais, "}
data: {"delta": "os produtos com maior risco são: "}
data: {"delta": "Coca-Cola 350ml (estoque: 4, mínimo: 10)"}
data: {"done": true, "sources": ["ai_logs/uuid", "stock_levels/uuid"]}
RAG Pipeline
1. Embed message  →  text-embedding-3-small (OpenRouter)
2. pgvector cosine similarity  →  ai_embeddings table  →  top-K=5 chunks
3. Augmented prompt  →  LLM via OpenRouter
4. Stream response tokens as SSE deltas

Indexed data: products + stock levels · agent reports · financial movements · sales history.

Webhooks

Method
Path
Role
Description
POST
/webhooks/supabase
internal
Supabase DB Webhook receiver. Dispatches correct Deep Agent based on table + event.
Auth
Header: x-webhook-secret: <SUPABASE_WEBHOOK_SECRET>
Validated against env var — NOT a user JWT.
Request Body (Supabase Webhook format)
{
  "type": "INSERT",
  "table": "movements",
  "record": { "id": "uuid", "product_id": "uuid", "type": "saida", ... },
  "schema": "public"
}
Dispatch Logic
table=movements  (INSERT|UPDATE)  →  EstoqueAgent
table=sales      (INSERT)          →  ComercialAgent
table=transfers  (INSERT)          →  LogisticsAgent
Response 202
{ "dispatched": "EstoqueAgent", "event_id": "uuid" }

Agent runs asynchronously. Response acknowledges dispatch, not completion.

Error Responses

400
Bad Request — validation error. Body: { "detail": [{ "loc": [...], "msg": "..." }] }
401
Unauthorized — missing or invalid JWT.
403
Forbidden — authenticated but role insufficient (e.g., operator hitting admin route).
404
Not Found — resource does not exist or is outside caller's company scope (RLS).
422
Unprocessable — FastAPI/Pydantic schema mismatch.
500
Internal Server Error — unexpected failure. Agent dispatch errors return 202 + async error written to ai_logs.
