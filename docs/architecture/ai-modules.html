Design Decisions

  • HR is out of scope for agents. Team management (create users, assign Operator/Admin roles) is a Settings page — no dedicated agent, no AI needed.
  • Event-driven, not user-triggered. Deep Agents run autonomously via Supabase DB webhooks (on data changes) or pg_cron schedules — frontend only consumes results via Realtime subscription.
  • Agent decoupling via Supabase. ComercialAgent writes its output to ai_logs; EstoqueAgent and SupplyChainAgent read it from there. No direct inter-agent calls.
  • RAG is separate from Deep Agents. Chat Q&A requires low latency — pgvector retrieval + LLM is faster and simpler than spawning an agent. Deep Agents handle background analysis, RAG handles foreground conversation.
  • All agent outputs are structured JSON saved to ai_logs table. Frontend queries and subscribes to this table — no agent-specific API endpoints needed.
