# Task 1: Schema de lotes e view de estoque atual

**Estimativa:** ~35 min  
**Dev:** Backend Dev 2  
**O que é e por que existe:**  
Lotes permitem rastrear validade e fornecedor por entrada. A view `estoque_atual` agrega movimentações para exibir quantidade em tempo real.

**Arquivos:**
- Criar: `supabase/migrations/<timestamp>_lotes_e_estoque.sql`

---

### Conceito: View vs Tabela Materializada

> View recalcula na hora — sempre atual, mas mais lenta em volumes grandes.  
> Tabela materializada é mais rápida, mas precisa de refresh manual.  
> Para Stocky (PME, volume baixo), view simples é suficiente.

<details>
<summary>💡 Solução — Passo 3</summary>

```sql
CREATE OR REPLACE VIEW estoque_atual AS
SELECT
  produto_id,
  SUM(
    CASE
      WHEN tipo = 'entrada' THEN quantidade
      WHEN tipo = 'saida'   THEN -quantidade
      ELSE 0
    END
  ) AS quantidade_atual
FROM movimentacoes
GROUP BY produto_id;
```

</details>
