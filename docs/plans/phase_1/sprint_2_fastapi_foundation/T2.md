# Task 2: Configurar cliente Supabase + settings

**Estimativa:** ~35 min  
**Dev:** Backend Dev 1  
**O que é e por que existe:**  
Centralizar configurações e o cliente Supabase em um singleton evita reconexões desnecessárias e garante que credenciais são lidas do `.env` em um único lugar.

**Arquivos:**
- Editar: `app/config.py`
- Editar: `app/database.py`
- Criar: `.env` (cópia de `.env.example` do Sprint 1)

---

### Conceito: Pydantic Settings

> `pydantic-settings` lê variáveis do `.env` automaticamente e valida tipos.  
> Se `SUPABASE_URL` estiver ausente no `.env`, a aplicação falha ao iniciar com erro claro.  
> Sem isso, erros de configuração aparecem só em runtime quando a primeira requisição é feita.

---

### Documentação
- [`pydantic-settings`](https://docs.pydantic.dev/latest/concepts/pydantic_settings/) — BaseSettings, leitura de .env
- [`supabase.create_client()`](https://supabase.com/docs/reference/python/initializing) — inicialização do cliente Python

# app/config.py
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    supabase_url: str
    supabase_anon_key: str
    supabase_service_role_key: str

    model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8")

settings = Settings()
```

# app/main.py
from contextlib import asynccontextmanager
from fastapi import FastAPI
from app.database import supabase_client

@asynccontextmanager
async def lifespan(app: FastAPI):
    result = supabase_client.table("produtos").select("count", count="exact").execute()
    print(f"Supabase conectado. Produtos no banco: {result.count}")
    yield

app = FastAPI(title="Stocky API", version="0.1.0", lifespan=lifespan)
```
