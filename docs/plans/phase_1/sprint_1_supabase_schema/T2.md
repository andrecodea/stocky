**Arquivos:**
- Criar: `supabase/migrations/<timestamp>_initial_schema.sql`

---

### Conceito: Migrations versionadas

> Migrations são arquivos SQL numerados que descrevem a evolução do schema.  
> O Supabase CLI aplica migrations em ordem e controla o que já foi rodado.  
> Sem isso, mudanças de schema ficam no dashboard e nenhum dev consegue reproduzir o banco localmente.

---

### Conceito: UUID como PK padrão

> Supabase usa `uuid` como chave primária padrão (`gen_random_uuid()`).  
> Isso evita colisões em sistemas distribuídos e não expõe contagem de registros.  
> Sem isso, IDs sequenciais (`1, 2, 3...`) vazam informação de volume para clientes.

---

```sql
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  nome            text NOT NULL,
  sku             text UNIQUE,
  descricao       text,
  preco_custo     numeric(10,2) NOT NULL DEFAULT 0,
  preco_venda     numeric(10,2) NOT NULL DEFAULT 0,

  estoque_minimo  integer NOT NULL DEFAULT 0,
```

<details>
<summary>💡 Solução — Passo 3</summary>

```sql
CREATE TYPE tipo_movimentacao AS ENUM ('entrada', 'saida', 'ajuste');

CREATE TABLE IF NOT EXISTS movimentacoes (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  produto_id      uuid NOT NULL REFERENCES produtos(id) ON DELETE RESTRICT,
  tipo            tipo_movimentacao NOT NULL,
  quantidade      integer NOT NULL,
  custo_unitario  numeric(10,2),
  observacao      text,
  usuario_id      uuid REFERENCES auth.users(id),
  criado_em       timestamptz NOT NULL DEFAULT now()
);
```

```sql
CREATE TYPE role_usuario AS ENUM ('operador', 'admin');

CREATE TABLE IF NOT EXISTS perfis (
  id        uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  nome      text NOT NULL,
  role      role_usuario NOT NULL DEFAULT 'operador',
  criado_em timestamptz NOT NULL DEFAULT now()
);
```
