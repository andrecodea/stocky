# Sprint 1 — Supabase Schema Foundation

**Dev:** Backend Dev 2 (Banco de Dados)  
**Duração estimada:** ~1 semana  
**Pré-requisito:** Conta Supabase criada, acesso ao projeto

## Objetivo

Configurar o projeto Supabase, criar o schema inicial do banco e configurar Auth com roles básicos. Ao final deste sprint, o banco está pronto para receber o cliente da API (Sprint 2).

## Contrato de saída

Ao concluir o sprint, o Backend Dev 1 (Sprint 2) deve conseguir:
- Conectar ao Supabase com service key
- Fazer queries nas tabelas `produtos`, `usuarios`, `movimentacoes`
- Criar/autenticar usuário via Supabase Auth com role `operador` ou `admin`
