# Task 3: Configurar Auth + RLS base

**Estimativa:** ~35 min  
**Dev:** Backend Dev 2  
**O que é e por que existe:**  
Ativar Row Level Security (RLS) nas tabelas criadas e definir as políticas mínimas por role. Sem RLS, qualquer usuário autenticado consegue ler/escrever dados de outros usuários — isso é um bug de segurança crítico.

**Arquivos:**
- Criar: `supabase/migrations/<timestamp>_rls_base.sql`

---

### Conceito: Row Level Security (RLS)

> RLS são regras no banco que determinam quais linhas cada usuário pode ler ou modificar.  
> O Supabase aplica RLS automaticamente quando ativado — sem precisar de lógica no backend.  
> Sem isso, um operador consegue acessar dados de qualquer empresa no banco, causando vazamento de dados.

---

### Conceito: `auth.uid()` e `auth.role()`

> `auth.uid()` retorna o UUID do usuário autenticado na sessão atual.  
> Para verificar o role customizado, é necessário consultar a tabela `perfis` onde o role está armazenado.  
> Sem isso, as políticas não conseguem diferenciar operador de admin.

```sql
ALTER TABLE produtos ENABLE ROW LEVEL SECURITY;

CREATE POLICY "produtos_select" ON produtos
  FOR SELECT TO authenticated USING (true);

CREATE POLICY "produtos_admin_write" ON produtos
  FOR ALL TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM perfis
      WHERE perfis.id = auth.uid()
        AND perfis.role = 'admin'
    )
  )
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM perfis
      WHERE perfis.id = auth.uid()
        AND perfis.role = 'admin'
    )
  );
```

<summary>💡 Solução — Passo 3</summary>

```sql
ALTER TABLE movimentacoes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "movimentacoes_select" ON movimentacoes
  FOR SELECT TO authenticated
  USING (
    usuario_id = auth.uid()
    OR EXISTS (
      SELECT 1 FROM perfis
      WHERE perfis.id = auth.uid() AND perfis.role = 'admin'
    )
  );

CREATE POLICY "movimentacoes_insert" ON movimentacoes
  FOR INSERT TO authenticated
  WITH CHECK (usuario_id = auth.uid());

CREATE POLICY "movimentacoes_admin_write" ON movimentacoes
  FOR UPDATE TO authenticated
  USING (
    EXISTS (SELECT 1 FROM perfis WHERE perfis.id = auth.uid() AND perfis.role = 'admin')
  );
```

<summary>💡 Solução — Passo 4</summary>

```sql
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.perfis (id, nome, role)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'nome', 'Usuário'),
    COALESCE((NEW.raw_user_meta_data->>'role')::role_usuario, 'operador')
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```
