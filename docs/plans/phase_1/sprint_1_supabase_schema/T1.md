---

### Conceito: Supabase CLI vs Dashboard

> O Supabase CLI permite versionar migrations como código SQL em `supabase/migrations/`.  
> O Dashboard permite mudanças manuais, mas essas não ficam no controle de versão.  
> Sempre usar o CLI para mudanças de schema — sem isso, o time não consegue reproduzir o banco localmente.

---

### Documentação
- [`supabase init`](https://supabase.com/docs/reference/cli/supabase-init) — inicializa projeto local
- [`supabase link`](https://supabase.com/docs/reference/cli/supabase-link) — conecta projeto local ao remote
- [`supabase db pull`](https://supabase.com/docs/reference/cli/supabase-db-pull) — puxa schema atual do remote
